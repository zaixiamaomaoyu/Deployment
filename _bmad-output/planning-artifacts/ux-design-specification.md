**作者:** maomaoyu

## Executive Summary

### Project Vision

**前端部署学习平台**是一个面向前端开发者的交互式学习产品，旨在帮助零部署经验的开发者从「第 0 步」开始掌握前端包部署技能。

**核心差异化：** 填补「第 0 步」空白——在用户动手之前，先帮他们理清「我需要准备什么、我该选哪个方案」。

**成功标志：** 用户说「这个网站太棒了，我第一次觉得部署没那么难，终于有人从我的起点开始教了」并推荐给朋友。

### Target Users

**用户 1：小林（入门开发者）**
- 工作 0.5-1 年，有前端开发能力
- 从未部署过项目，不知道需要准备什么
- 看到教程中「SSH 连接服务器」就懵了
- 需要：从「第 0 步」开始的引导、术语解释、即时帮助

**用户 2：张工（进阶开发者）**
- 工作 3+ 年，用过 Vercel 等免费平台
- 知识零散，遇到问题只会搜索复制粘贴
- 想系统学习服务器部署、域名配置等
- 需要：技能评估、系统化学习路径、深度诊断

### Key Design Challenges

1. **决策树交互设计**
   - 如何让问答流程自然、不枯燥？
   - 如何处理用户「不确定」的情况？
   - 如何展示推荐结果的置信度？

2. **AI 助手对话体验**
   - 如何让流式输出体验流畅？
   - 如何区分「简单问答」和「深度诊断」？
   - 如何处理 AI 响应延迟或错误？

3. **知识内容导航**
   - 如何让 6×5=30 个知识点易于浏览？
   - 如何展示学习进度和技能水平？
   - 如何引导用户找到下一步该学什么？

### Design Opportunities

1. **差异化起点**
   - 从「不知道要准备什么」开始，而非假设已有环境
   - 决策树可以是产品的核心差异化体验

2. **陪伴式学习**
   - AI 助手主动推进学习进度
   - 不是被动问答，而是引导式教练

3. **游戏化激励（Vision 阶段）**
   - 技能雷达图可视化
   - 成就徽章、学习进度追踪

## Core User Experience

### Defining Experience

**核心交互：决策树 + AI 助手**

用户通过问答式决策树找到适合自己的部署方案，过程中可随时向 AI 助手提问。这是产品的核心差异化体验。

### Platform Strategy

| 项目 | 决策 |
|-----|------|
| 平台 | Web 应用（响应式设计） |
| 交互方式 | 鼠标/键盘为主，触屏兼容 |
| 浏览器支持 | Chrome、Edge、Firefox、Safari 最新两个版本 |
| 移动端优先级 | 桌面优先，移动端适配 |

### Effortless Interactions

**应该自然的操作：**
- 决策树导航 — 点击即前进
- AI 对话 — 像聊天一样提问
- 复制命令 — 一键复制

**应该自动完成的：**
- 学习进度保存
- 对话历史保留
- 下一步推荐

### Critical Success Moments

| 时刻 | 用户感受 |
|-----|---------|
| 决策树完成 | 「原来我适合这个方案！」|
| 首次部署成功 | 「我做到了！」|
| AI 解答问题 | 「终于有人讲明白了」|

### Experience Principles

1. **从零开始** — 不假设先备知识
2. **即时反馈** — 每个操作有响应
3. **渐进引导** — 先给答案再解释
4. **容错设计** — 允许随时回退

## Design System Foundation

### 设计系统选择

**Element Plus** — 基于 Vue 3 的企业级 UI 组件库

### 选择理由

| 因素 | 说明 |
|-----|------|
| 技术栈匹配 | PRD 已锁定 Vue 3 + Element Plus |
| 组件丰富度 | 60+ 企业级组件，覆盖大部分需求 |
| 中文支持 | 完善的中文文档和社区 |
| 可定制性 | 支持 CSS 变量主题定制 |
| 无障碍 | 内置 ARIA 支持 |

## User Journey Flows

### 旅程 1: 新用户首次使用决策树

```
首页 → 点击「开始选择方案」→ 决策树问答
     ↓
     Q1: 有服务器吗？ → 没有
     ↓
     Q2: 想花钱吗？ → 不想
     ↓
     Q3: 项目类型？ → 静态网站
     ↓
     推荐结果: Vercel/Netlify
     ↓
     查看步骤指南 → AI 助手辅助 → 完成部署
```

### 旅程 2: 用户向 AI 提问

```
任意页面 → 点击 AI 助手按钮
        ↓
        输入问题 → AI 流式回复
        ↓
        复制代码/继续追问 → 解决问题
```

## Accessibility Strategy

### WCAG 2.1 AA 合规

| 项目 | 要求 |
|-----|------|
| 颜色对比度 | 文本 4.5:1，大文本 3:1 |
| 键盘导航 | 所有交互可通过键盘完成 |
| 焦点指示 | 可见的焦点状态 |
| ARIA 标签 | 所有交互元素有清晰标签 |
| 错误提示 | 表单错误有明确说明 |

### 关键实现

```vue
<!-- 决策树选项可访问性 -->
<button
  role="radio"
  :aria-checked="isSelected"
  aria-label="选项：有服务器"
>
  有服务器
</button>

<!-- AI 消息区域 -->
<div
  role="log"
  aria-live="polite"
  aria-label="AI 助手回复"
>
  {{ message }}
</div>
```
